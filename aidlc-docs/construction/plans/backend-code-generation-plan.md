## Unit Context & Design Summary

### Stories Assigned to Backend Unit
From `unit-of-work-story-map.md`, the Backend unit is responsible for:
- US-1.1: User Registration and Authentication
- US-1.2: Password Reset and Account Management
- US-2.1: Create Tournament and Session
- US-2.2: Manage Scoring Rules
- US-3.1: Real-time Score Updates
- US-3.2: Arrow Detection and Validation
- US-3.3: Leaderboard Updates
- US-4.1: Generate Tournament Reports
- US-5.1: Camera Connection Management
- US-5.2: Image Capture and Storage
- US-6.1: Performance Optimization (Caching)
- US-6.2: Database Resilience
- US-7.1: User Activity Audit Logging
- US-7.2: Security Baseline Implementation

### Domain Entities (From Functional Design)
1. **User** - username, email, password_hash, role, created_at, updated_at
2. **Tournament** - name, location, start_date, end_date, created_by_user_id
3. **Session** - tournament_id, name, status (active/paused/completed), created_at
4. **SessionArcher** - session_id, archer_id, current_round, total_score
5. **Score** - session_archer_id, round, arrow, zone, points, image_id, validated_by_ai
6. **Camera** - name, camera_type (USB/RTSP/HTTP), url, lane_id, status (connected/disconnected)
7. **CameraLaneAssignment** - camera_id, lane, session_id, assigned_at

### Services (From Functional Design)
1. **AuthService** - User registration, login, token validation, password reset
2. **ScoringService** - Score calculation, validation, arrow detection pipeline
3. **CameraService** - Camera connection, image capture, reconnection logic
4. **ReportService** - Report generation (PDF, CSV, JSON)
5. **LeaderboardService** - Real-time leaderboard aggregation and caching
6. **ImageProcessingService** - Computer vision: color detection, edge detection, ML-based detection

### Dependencies & Integration
- **Frontend**: REST API endpoints, WebSocket for real-time updates
- **Database**: PostgreSQL with SQLAlchemy ORM
- **Cache**: Redis for leaderboards, session data, camera status
- **Message Queue**: In-process EventBus (EventPublisher) for MVP
- **External**: Optional reports storage (S3 in production)

### Tech Stack (Locked from Previous Stages)
- **Language**: Python 3.11+
- **Framework**: FastAPI + Uvicorn
- **ORM**: SQLAlchemy 2.0+
- **Database**: PostgreSQL 15+
- **Cache**: Redis 7+
- **Testing**: pytest + pytest-asyncio
- **Validation**: Pydantic v2
- **API Documentation**: OpenAPI (auto-generated by FastAPI)
- **Logging**: structlog (JSON structured logs)
- **Image Processing**: OpenCV 4.8+, Pillow
- **Email**: python-multipart (form handling)
- **Auth**: PyJWT + passlib + bcrypt
- **Type Hints**: mypy-compatible

### NFR Design Patterns to Implement
From `nfr-design-patterns.md`, these 20 patterns must be implemented in code:
1. Database Connection Resilience (retry with exponential backoff)
2. Scoring Failure Recovery (automatic retry with notification)
3. WebSocket Disconnection Handling (30s grace period)
4. Image Fallback Chain (color → edge → ML voting)
5. Camera Reconnection (retry(5) with user notification)
6. Session Isolation (per-tournament concurrency)
7. Adaptive Load Balancing (ThreadPool CPU-based scaling)
8. Query Optimization (connection pooling, batch operations)
9. Storage Management (90-day rotation, quota enforcement)
10. ThreadPool Scaling (4 base, 8 max workers, CPU monitoring)
11. Application Caching (TTLCache for cameras/sessions/leaderboards)
12. Image Compression (JPEG 70% quality for 20-30% latency reduction)
13. Leaderboard Caching (1-min TTL, event-driven invalidation)
14. Connection Pool Tuning (min 5, max 20 adaptive connections)
15. WebSocket Batching (100ms window or 10 events)
16. Input Validation (Pydantic BaseModel + @validator)
17. Rate Limiting (1000 req/min per IP)
18. Audit Logging (structlog + audit_logs table)
19. Secret Management (os.getenv for DATABASE_URL, JWT_SECRET)
20. CORS Configuration (allow-all for internal/trusted networks)

### Logical Components to Implement
From `logical-components.md`, these 10 components must be implemented:
1. **PostgreSQL + Patroni** - Database with failover
2. **Redis** - Distributed cache (512MB, allkeys-lru)
3. **Filesystem Storage** - /storage/raw/, /storage/annotated/, /storage/archives/
4. **ThreadPool** - ThreadPoolExecutor(4-8 workers, CPU-based)
5. **EventBus** - In-process pub/sub event publishing
6. **CacheManager** - TTL-based cache with invalidation
7. **LoadManager** - Monitor CPU/memory, auto-scale pools
8. **Structured Logging** - structlog JSON to stdout + audit_logs table
9. **WebSocketManager** - Real-time broadcasts with 30s grace period

### Phase 3: Service Layer (Business Logic)

**Step 9: Create AuthService (src/services/auth_service.py)**
- [ ] register_user(username, email, password) → User
  - Validate username/email uniqueness (query first, raise if exists)
  - Hash password with bcrypt, generate salt
  - Create user record in database
  - Return user DTO
- [ ] login_user(username, password) → {access_token, refresh_token}
  - Query user by username
  - Verify password with bcrypt
  - Generate JWT access token (8h expiration) + refresh token (30d)
  - Return tokens
- [ ] refresh_access_token(refresh_token) → access_token
  - Decode refresh token, validate expiration
  - Generate new access token
- [ ] verify_token(token) → user_id
  - Decode JWT, validate signature + expiration
  - Return user_id
- [ ] reset_password(user_id, new_password) → bool
  - Hash new password, update user record
- [ ] Unit Tests: test_register_user (success, duplicate email, weak password), test_login_user (success, invalid password, user not found), test_token_refresh, test_password_reset
- [ ] Story Coverage: US-1.1, US-1.2

**Step 10: Create ScoringService (src/services/scoring_service.py)**
- [ ] validate_score(zone: int, points: int) → bool
  - Check zone in [0-10], points in [0-10], zone-to-points mapping valid
- [ ] start_round(session_archer_id) → RoundState
  - Initialize new round, set status=active, increment current_round
  - Return round metadata
- [ ] record_score_with_retry(session_archer_id, round, arrow, zone, points, image_id) → Score
  - Implement retry logic (max 2 attempts, exponential backoff)
  - Call _validate_score (business rules)
  - Save to database, catch OperationalError (DB timeout) → retry
  - Return score record
  - Emit event: ScoreCalculated (for leaderboard/websocket)
  - If retry fails, raise ScoringFailedError
- [ ] calculate_total_score(session_archer_id) → int
  - Sum all points for archer in session
  - Cache result in Redis (TTL 1m)
  - Return total
- [ ] get_leaderboard(session_id) → List[LeaderboardItem]
  - Order by total_score DESC, by archer name
  - Return top N archers
  - Implement caching (TTL 1m, invalidate on ScoreCalculated event)
- [ ] Unit Tests: test_validate_score, test_record_score_success, test_record_score_with_retry, test_calculate_total_score, test_leaderboard_caching
- [ ] Story Coverage: US-2.2, US-3.1, US-3.2, US-3.3

**Step 11: Create CameraService (src/services/camera_service.py)**
- [ ] connect_camera(camera_id) → Camera
  - Attempt connection based on camera_type (USB/RTSP/HTTP)
  - Set status=connected, update last_connected_at
  - Return camera record
  - Emit event: CameraConnected
- [ ] reconnect_camera_with_retry(camera_id, max_retries=5) → Camera | None
  - Implement retry loop with exponential backoff (2^n seconds)
  - On attempt 3, notify user: "Camera reconnection in progress"
  - On success, emit CameraConnected event
  - On failure, emit CameraDisconnected event, return None
  - Story Coverage: NFR Pattern #5
- [ ] disconnect_camera(camera_id) → bool
  - Set status=disconnected
  - Emit event: CameraDisconnected
- [ ] get_assigned_camera(session_id, lane) → Camera
  - Query CameraLaneAssignment for session + lane
  - Return camera record
  - Raise CameraNotAssignedError if not found
- [ ] Unit Tests: test_connect_camera (success, failure), test_reconnect_with_retry (success after N retries), test_disconnect_camera
- [ ] Story Coverage: US-5.1, US-5.2

**Step 12: Create ImageService (src/services/image_service.py)**
- [ ] Implement NFR Pattern #4: Image Fallback Chain (color → edge → ML voting)
- [ ] detect_arrow_in_image(image_path) → {zone: int, confidence: float}
  - Method 1 (color detection): Use OpenCV HSV range detection for arrow color
  - Method 2 (edge detection): Use Canny edge detection + contour analysis
  - Method 3 (ML-based): Call ML model (pre-trained or placeholder)
  - Voting logic: if confidence > threshold, return result; else try next method
  - Fallback: if all fail, return {zone: None, confidence: 0, method: "failed"}
- [ ] preprocess_image(image_data) → image
  - Convert to OpenCV format
  - Resize to standard dimensions
  - Apply JPEG compression (quality=70) → NFR Pattern #12
- [ ] save_image(image_data, session_id, round, arrow) → image_id
  - Generate image_id UUID
  - Save to /storage/raw/{session_id}/{image_id}.jpg
  - Check storage quota (10GB), raise StorageQuotaExceeded if over
  - Return image_id
- [ ] archive_old_images(days=90) → int
  - Find images older than 90 days
  - Compress to tar.gz, move to /storage/archives/
  - Return count archived
  - Emit event: StorageArchived
- [ ] Unit Tests: test_detect_arrow_color, test_detect_arrow_edge, test_detect_arrow_ml, test_image_preprocessing, test_save_image, test_storage_quota_enforcement
- [ ] Story Coverage: US-3.2, US-5.2

**Step 13: Create LeaderboardService (src/services/leaderboard_service.py)**
- [ ] get_leaderboard_cached(session_id) → List[LeaderboardItem]
  - Implement TTLCache pattern: check Redis for key `leaderboard:{session_id}`
  - If cache hit, return cached result
  - If cache miss, query database (SUM(points) GROUP BY archer_id ORDER BY total DESC)
  - Cache result in Redis (TTL 1m)
  - Return leaderboard
- [ ] invalidate_leaderboard(session_id) → bool
  - Delete Redis key `leaderboard:{session_id}`
  - Called when ScoreCalculated event published
- [ ] Unit Tests: test_leaderboard_caching (hit, miss, invalidation), test_leaderboard_ordering
- [ ] Story Coverage: US-3.3, US-6.1

**Step 14: Create ReportService (src/services/report_service.py)**
- [ ] generate_report(session_id, format: pdf|csv|json) → bytes
  - Query session data + all scores + leaderboard
  - Format data based on format parameter:
    - PDF: Use reportlab or similar (title, tournament info, leaderboard table, scores table)
    - CSV: Generate CSV rows (archer, score, round, arrow, zone, points)
    - JSON: Return structured JSON object
  - Save to /storage/reports/{session_id}/{timestamp}.{format}
  - Return file bytes
- [ ] Unit Tests: test_generate_pdf_report, test_generate_csv_report, test_generate_json_report
- [ ] Story Coverage: US-4.1

**Step 15: Create HealthService (src/services/health_service.py)**
- [ ] check_database_health() → bool
  - Execute SELECT 1 query
  - Return True if successful, False if timeout
- [ ] check_cache_health() → bool
  - Execute Redis PING
  - Return True if responsive, False if timeout
- [ ] check_storage_health() → {available_gb, used_gb, quota_gb}
  - Check filesystem /storage directory
  - Calculate disk usage
  - Return usage metrics
- [ ] check_threadpool_health() → {active, max, utilization_percent}
  - Get ThreadPoolExecutor active count + max_workers
  - Calculate utilization %
  - Return metrics
- [ ] get_system_health() → {status: ok|degraded|down, timestamp, components: {database, cache, storage, threadpool}}
  - Call all health checks
  - Aggregate status (all ok → ok, any failing → degraded, all failing → down)
  - Return full health report
- [ ] Story Coverage: US-6.2

### Phase 4: API Layer (FastAPI Routes)

**Step 16: Create Auth Routes (src/api/auth.py)**
- [ ] POST /auth/register
  - Body: {username, email, password, password_confirm}
  - Validate passwords match, password strength
  - Call AuthService.register_user()
  - Return: {user_id, username, email}
  - Status: 201 Created on success, 400 Bad Request on validation error, 409 Conflict on duplicate email

  - Body: {username, password}
  - Call AuthService.login_user()

  - Body: {refresh_token}
  - Call AuthService.refresh_access_token()
  - Return: {access_token, token_type: Bearer}
  - Status: 200 OK on success, 401 Unauthorized on invalid refresh token
- [ ] POST /auth/reset-password
  - Body: {user_id, old_password, new_password}
  - Verify old_password, call AuthService.reset_password()
  - Return: {success: true}
  - Status: 200 OK, 401 Unauthorized
- [ ] Unit Tests: test_register_endpoint, test_login_endpoint, test_refresh_endpoint
- [ ] Story Coverage: US-1.1, US-1.2

**Step 17: Create Tournament Routes (src/api/tournaments.py)**
- [ ] GET /tournaments
  - Query params: skip=0, limit=10, search=optional
  - Return: List[TournamentResponse]
